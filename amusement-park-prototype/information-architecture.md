# 游乐场管理系统 - 信息架构设计

## 设计原则
1. **易用性优先**：减少点击深度，增加快捷入口
2. **角色化设计**：不同用户看到不同功能集
3. **数据驱动**：核心指标可视化展示
4. **一致性**：统一的交互模式和视觉风格

## 用户角色与权限
| 角色 | 核心需求 | 功能权限 |
|------|----------|----------|
| **运营人员** | 日常票务、会员管理 | 票务管理、会员管理、商品销售 |
| **店长** | 整体运营监控、报表 | 实时监控、经营报表、员工管理 |
| **仓管人员** | 商品库存管理 | 商品管理、库存盘点、采购管理 |
| **财务人员** | 收支核对、财务报表 | 财务管理、对账、报表导出 |
| **老板/管理员** | 全局监控、系统设置 | 所有功能 + 系统设置、权限管理 |

## 首页仪表盘模块设计（基于用户提供的首页图片）

### 核心数据卡片（顶部）
1. **今日营收** - 金额、同比、环比
2. **入园人数** - 实时人数、各时段分布
3. **会员新增** - 今日新增、会员总数
4. **设备状态** - 正常运行数/总数、报警数

### 图表区域（中部）
1. **营收趋势图** - 近7天/30天营收曲线
2. **客流热力图** - 各时段入园人数分布
3. **会员增长图** - 新增会员趋势
4. **设备状态图** - 设备运行状态分布

### 快捷操作区（右侧）
1. **快速售票** - 一键进入售票界面
2. **会员办卡** - 快速会员注册
3. **商品上架** - 新品快速上架
4. **设备报修** - 设备故障上报

### 待办事项（底部）
1. **待处理订单** - 需要确认的订单
2. **设备报警** - 需要处理的设备问题
3. **会员审批** - 待审核的会员申请
4. **财务对账** - 需要核对的账目

## 页面布局规范

### 通用布局结构
```
┌─────────────────────────────────────────┐
│ 顶部导航栏 (一级菜单)                   │
├─────────────────────────────────────────┤
│ 左侧导航栏 (二级菜单) │ 内容区 (三级菜单)│
│                       │                 │
│                       │                 │
│                       │                 │
└─────────────────────────────────────────┘
```

### 响应式设计
- **桌面端**：左侧导航固定宽度，内容区自适应
- **平板端**：左侧导航可收起/展开
- **移动端**：底部标签导航，内容全屏

## 交互设计规范

### 导航交互
1. **菜单hover效果**：背景色变化 + 轻微放大
2. **菜单激活状态**：高亮显示 + 左侧指示条
3. **面包屑导航**：显示当前位置路径
4. **快捷收藏**：支持将常用功能收藏到首页

### 数据展示交互
1. **卡片hover**：轻微上浮 + 阴影加深
2. **图表hover**：显示详细数据点
3. **数据刷新**：平滑过渡动画
4. **加载状态**：骨架屏 + 进度指示

### 操作反馈
1. **成功操作**：右上角toast提示
2. **错误操作**：红色边框 + 错误提示
3. **确认操作**：二次确认弹窗
4. **批量操作**：选中项计数显示
